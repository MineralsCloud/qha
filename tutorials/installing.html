2.2.1. Compatibility¶

qha is compatible with Python 3.5 or later, and Numpy versions 1.7 to 1.14.

Our supported platforms are:

  • Linux x86 (32-bit and 64-bit)
  • Windows 7 and later (32-bit and 64-bit)
  • OS X 10.9 and later (64-bit)
  • NVIDIA GPUs of compute capability 2.0 and later
  • AMD APUs supported by the HSA 1.0 final runtime (Kaveri, Carrizo)

2.2.2. Dependencies¶

  • bigfloat [1]
  • lazy-property
  • matplotlib [2]
  • Numba
  • NumPy
  • pandas
  • PyYAML
  • scientific-string
  • SciPy
  • text-stream

Notes:

[1]GMP and MPFR libraries are required to use bigfloat package. On macOS, install these libraries via brew install mpfr; on Linux, install libmpfr-dev , for example, on Ubuntu use [sudo] apt-get install libmpfr-dev; on Windows, bigfloat can be installed from the binary file, please check Unofficial Windows Binaries for Python Extension Packages
[2]For some system, python-tkinter package is needed by matplotlib, otherwise the plot function will not work.

2.2.3. Installing the qha package¶

2.2.3.1. Installing using Conda¶

The easiest way to install qha and get updates is by using Conda, a cross-platform package manager and software distribution maintained by Anaconda, Inc. You can either use Anaconda to get the full stack in one download, or Miniconda which will install the minimum packages needed to get started.

Once you have conda installed, just type:

$ conda install qha

or:

$ conda update qha

2.2.3.2. Installing using PyPI¶

qha can be installed via pip from PyPI.:

$ pip install qha

This will likely require the installation of a number of dependencies, including NumPy, will require a compiler to compile required bits of code, and can take a few minutes to complete.

2.2.3.3. Installing from source¶

Download the latest release , and go to the top-level directory, run:

$ pip install -e .
